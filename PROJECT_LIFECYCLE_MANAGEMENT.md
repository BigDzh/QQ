## 1. 项目概述

### 1.1 项目背景

QQ_EXPORT 是一个综合的项目管理系统，提供项目管理、模块管理、组件管理、软件管理、借入借出管理、审计日志、备份管理等功能。

### 1.2 技术栈

| 层级 | 技术 |
|------|------|
| 前端框架 | React 18 + TypeScript |
| 路由 | React Router v6 |
| 状态管理 | React Context |
| UI 组件 | Tailwind CSS + Lucide Icons |
| 数据存储 | IndexedDB (Dexie.js) |
| 测试框架 | Vitest + React Testing Library |
| E2E 测试 | Playwright |
| 构建工具 | Vite |
| 开发环境 |Electron (可选) |

### 1.3 当前项目状态

| 指标 | 当前值 | 目标值 |
|------|--------|--------|
| 构建状态 | ✅ 通过 | 100% |
| 测试通过率 | 100% (278/278) | >95% |
| TypeScript 警告 | ~200个 | <50个 |
| 代码覆盖 | 待评估 | >70% |

## 3. 项目维护机制

### 3.1 问题跟踪

#### 3.1.1 问题分类

| 类别 | 说明 | 处理时限 |
|------|------|----------|
| P0 - 紧急 | 系统崩溃、数据丢失 | 4小时 |
| P1 - 高 | 核心功能不可用 | 24小时 |
| P2 - 中 | 功能缺陷、性能问题 | 7天 |
| P3 - 低 | UI问题、改进建议 | 下一个迭代 |

#### 3.1.2 问题状态

```
新建 → 确认 → 分配 → 修复中 → 代码审查 → 测试 → 发布 → 关闭

 └───┴───┴───────┴───────┴─────────┴──────────┴──────┴──────┘
                              (可退回)
```
