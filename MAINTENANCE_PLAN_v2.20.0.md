## 五、代码审查机制

### 5.1 审查标准

#### 代码规范检查清单

| 检查项 | 标准 | 工具 |
|--------|------|------|
| TypeScript类型 | 严格模式 | tsc |
| ESLint规范 | 无警告 | eslint |
| 代码格式化 | Prettier规范 | prettier |
| 提交信息 | Conventional Commits | commitlint |

#### 审查流程

```
代码变更 → 本地测试 → ESLint检查 → TypeScript检查 → 代码审查 → 合并
```

### 5.2 审查要点

#### 功能审查
- [ ] 功能实现是否符合需求
- [ ] 边界条件是否处理
- [ ] 错误处理是否完善

#### 代码质量审查
- [ ] 是否有重复代码
- [ ] 是否有内存泄漏风险
- [ ] 是否有性能问题

#### 安全审查
- [ ] 是否有XSS风险
- [ ] 是否有敏感信息泄露
- [ ] 是否有SQL注入风险

### 5.3 审查责任

| 变更类型 | 审查要求 |
|----------|----------|
| Bug修复 | 至少自检 |
| 小改动 | 自检 + 同事检视 |
| 重大改动 | 团队评审 |
