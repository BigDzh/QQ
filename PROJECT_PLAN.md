## 1. 项目概述

### 1.1 项目定位
项目全生命周期管理系统（导出版）是一个基于 Web/Electron 的企业级应用，用于管理项目、模块、组件、软件、系统等全生命周期资源。

### 1.2 现有技术栈
| 类别 | 技术选型 |
|------|---------|
| 前端框架 | React 18.2.0 |
| 语言 | TypeScript 5.3.3 |
| 构建工具 | Vite 5.1.4 |
| 样式方案 | Tailwind CSS 3.4.1 |
| 路由 | React Router DOM 6.22.0 |
| 状态管理 | React Context |
| 图表 | Recharts 2.12.0 |
| 桌面打包 | Electron 28.2.0 |
| 单元测试 | Vitest 4.1.1 |
| E2E测试 | Playwright 1.58.2 |
| 包管理 | npm |

### 1.3 现有项目结构
```
src/
├── components/     # UI组件
├── context/        # React Context
├── hooks/          # 自定义Hooks
├── pages/          # 页面组件
├── services/       # 业务服务
├── types/          # TypeScript类型
├── utils/          # 工具函数
├── workers/       # Web Workers
└── test/          # 测试文件
```

## 5. 代码规范

### 5.1 TypeScript规范
- 启用严格模式 (`strict: true`)
- 所有组件使用 TypeScript 接口定义 props
- 避免使用 `any` 类型
- 使用路径别名 `@/` 引用 src 目录

### 5.2 React组件规范
- 使用函数式组件 + Hooks
- 组件文件使用 PascalCase 命名
- Props 接口命名: `{ComponentName}Props`
- 事件处理函数使用 `handle` 前缀

### 5.3 文件组织规范
| 类型 | 目录 | 命名规范 |
|------|------|---------|
| 页面组件 | pages/ | PascalCase |
| 业务组件 | components/ | PascalCase |
| UI组件 | components/ui/ | PascalCase |
| Hooks | hooks/ | camelCase, use前缀 |
| 服务 | services/ | camelCase |
| 工具函数 | utils/ | camelCase |
| 类型定义 | types/ | camelCase, .d.ts |

### 5.4 Git提交规范
```
type(scope): subject

# type: feat | fix | docs | style | refactor | test | chore
# scope: 模块名称
# subject: 简短描述
```

## 6. 版本控制策略

### 6.1 分支模型
- `main`: 主分支，稳定版本
- `develop`: 开发分支
- `feature/*`: 功能分支
- `fix/*`: 修复分支
- `release/*`: 发布分支

### 6.2 版本号规范
- 格式: `主版本.次版本.修订号`
- 主版本: 不兼容的API修改
- 次版本: 向后兼容的功能新增
- 修订号: 向后兼容的问题修复

### 6.3 当前版本状态
- 当前版本: v2.19.1-export
- 目标版本: v2.20.0-export（下一迭代）

## 7. 个人开发工作流

### 7.1 日常开发流程
1. 从 `develop` 创建功能分支
2. 在分支上开发功能
3. 本地测试通过后提交代码
4. 合并回 `develop` 分支

### 7.2 开发节奏建议
| 阶段 | 时长 | 内容 |
|------|------|------|
| 上午 | 2-3小时 | 深度开发（功能实现） |
| 下午 | 2-3小时 | 测试调试/文档 |
| 每日 | 30分钟 | 代码审查与优化 |

### 7.3 任务管理
- 使用 TODO 列表跟踪任务
- 优先完成 P0/P1 任务
- 记录工作中的问题和解决方案
